 API Documentation                       Introduction The API will provide end points to access different sections inside a CCDS as well as the full CCDS. The clinical sections and the full CCDS, are specific per doctor encounter and will be restricted by the date provided in the request. The request should include the patient and/or the date range. If no date range is provided in the request, the CCDS will be created with the data available up until the date when the request is made. The API, also provides a way to search for patients by using the first name, last name or date of birth.                         Sections Included The following sections will be available under request: 1. The sections “Patient Name”, “Gender”, “Date of Birth”, “Race”, “Ethnicity”, “Preferred Language” are considered demographic sections.  Clinical Sections  The clinical sections can be queried by date. The sections “Smoking Status”, “Problems”, “Medications”, “Medication Allergies”, “Laboratory Tests”, “Laboratory Results”, “Vital Signs”, “Procedures”, “Care Team Member”, “Immunizations”, “Implantable Devices”, “Plan of Treatment”, “Goals” and “Health Concerns” are considered clinical sections. If the date range is not specified in the request, the CCDS will be created with the data available up to when the request is made.   Demographic End Points  1.
